# Business Requirements Document

## Project Name
JourneyGuard

## Problem Statement
Customers need a reliable online platform to register, search journeys, apply railcards, book tickets, and cancel bookings seamlessly.

## Users
* **Customer**: Browses, books, and manages rail tickets.
* **Administrator**: Manages schedules, system settings, and data.
* **Support Agent**: Assists customers and resolves tickets.

## Main Features
1. Registration
2. Login
3. Search Journey
4. Book Ticket
5. Cancel Booking
6. Booking History
7. Railcard Management
8. Support Tickets

## Non-functional Requirements
* **Performance**: Fast response time.
* **Security**: Secure login and encrypted data.
* **Usability**: Mobile-friendly interface.
* **Availability**: Highly reliable uptime.
* **Maintainability**: Clean code structure, easy to maintain.
