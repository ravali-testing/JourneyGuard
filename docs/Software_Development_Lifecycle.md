# SDLC & STLC Notes

## Software Development Lifecycle (SDLC) Phases
1. Requirement Analysis
2. Planning
3. Design
4. Development
5. Testing
6. Deployment
7. Maintenance

## Software Testing Lifecycle (STLC) Phases
1. Requirement Analysis
2. Test Planning
3. Test Design
4. Environment Setup
5. Test Execution
6. Defect Reporting
7. Test Closure

## Conceptual Q&A

### What is SDLC?
SDLC (Software Development Lifecycle) is the end-to-end framework that defines the entire process of building software from scratch, starting from the initial idea and requirements gathering all the way to deployment and continuous maintenance.

### What is STLC?
STLC (Software Testing Lifecycle) is a specific subset of the SDLC that focuses entirely on the testing process. It details the sequential phases required to ensure the software meets quality standards.

### How are they different?
* **Scope**: SDLC covers the entire development journey (including coding and infrastructure); STLC focuses *only* on verification and validation.
* **Timing**: SDLC starts the moment a project is conceived; STLC typically kicks off as soon as requirements are defined so testers can start planning before code is even written.
* **Goal**: SDLC's goal is to build a working software product; STLC's goal is to find defects and ensure the product works correctly.
